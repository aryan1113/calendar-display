    About this project (for curious users)

    Motivation: this page started as a practical way to quickly build and share a clean weekly timetable URL without account setup.

    Data source acknowledgement: schedule-appkp.netlify.app

    What felt difficult in that source app

    1. Courses were effectively limited to one subsection at a time.
    2. Batch-selection controls felt too small and easy to miss.
    3. Selection flow required checking small controls instead of selecting the full row.
    4. Timetable interaction felt less intuitive because day updates needed extra scrolling.
    5. Term timetable view felt redundant when course-time-venue metadata was stable across weeks.
    6. No straightforward way to share a timetable directly through URL state.
    7. Export to ICS then import into a calendar app felt cumbersome.

    Alternative discovered later

    iimk-pgp-sched.duckdns.org improves on both this project and the Netlify one in several ways:

    1. Supports direct PDF upload by users.
    2. Groups courses by area (LSM, Finance, DSOM, and others).
    3. Supports installable web-app flow for notifications.
    4. Has a subscription model for cancelled class, reschedule, room change, and morning summary alerts.

    Common ground

    1. Both approaches support editing a course-link style selection.
    2. Both support URL-based sharing, but this project encodes selected courses directly in the URL, while the DuckDNS app uses a unique session identifier.
